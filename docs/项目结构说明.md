## 📋 文件分类说明

### 根目录 - 主要工具

放置用户最常用的核心工具，方便快速访问：

- **01 低粉爆款挖掘 .html** - 搜索和发现优质低粉视频
- **02 YouTube博主管理工具.html** - 管理和追踪目标博主
- **API池管理工具.html** - 统一管理所有工具的API Key
- **youtube-tools-p5-refined.css** - 所有工具共用的主样式文件

### docs/ - 文档目录

集中存放所有说明文档和指南：

- **用户指南**：快速入门、使用说明、故障排除
- **开发文档**：设计系统、更新日志、迁移指南
- **API文档**：API配置、问题修复指南

### utilities/ - 辅助工具目录

存放诊断、修复和测试工具：

- **快速修复工具.html** - 一键诊断和修复常见问题
- **存储状态检查.html** - 查看数据存储状态
- **API池诊断修复工具.html** - 诊断和修复API池问题
- **设计预览.html** - 预览设计更新

### css/ - 样式目录

存放非核心样式文件：

- **design-system.css** - 设计系统样式
- **modern-theme.css** - 现代主题
- **notion-theme.css** - Notion风格主题

### 对标监控/ - 子项目目录

视频监控工具的独立子项目，包含其特定的HTML、CSS和JS文件。

## 🔗 文件引用规则

### 主工具 → CSS
```html
<!-- 根目录的主工具引用主CSS -->
<link rel="stylesheet" href="youtube-tools-p5-refined.css">
```

### 辅助工具 → CSS
```html
<!-- utilities/中的工具引用主CSS -->
<link rel="stylesheet" href="../youtube-tools-p5-refined.css">

<!-- 或引用css/中的样式 -->
<link rel="stylesheet" href="../css/modern-theme.css">
```

### 对标监控 → CSS
```html
<!-- 对标监控/中的工具引用主CSS -->
<link rel="stylesheet" href="../youtube-tools-p5-refined.css">
```

### 文档 → 工具
```markdown
<!-- docs/中的文档引用工具 -->
[快速修复工具](../utilities/快速修复工具.html)
[API管理](../API池管理工具.html)
```

### 工具 → 文档
```html
<!-- utilities/中的工具引用文档 -->
<a href="../docs/故障排除指南.md">查看指南</a>
```

## 🎯 优势说明

### 1. 清晰的分类
- 核心工具在根目录，用户直接打开
- 辅助工具在utilities/，不干扰主界面
- 所有文档集中在docs/，便于查找

### 2. 维护方便
- 文档修改不影响工具
- 样式文件独立管理
- 子项目保持独立性

### 3. 扩展性好
- 新增工具放入对应目录
- 新增文档统一放docs/
- 新增样式放css/

## 🔄 从旧结构迁移

如果你有旧版本的项目：

1. **主工具保持不变** - 已在根目录
2. **文档自动更新** - 路径已更新为相对路径
3. **辅助工具路径** - 从 `快速修复工具.html` 变为 `utilities/快速修复工具.html`
4. **文档路径** - 从 `README.md` 变为 `docs/README.md`

## 💡 使用建议

1. **日常使用** - 直接打开根目录的主工具
2. **遇到问题** - 使用utilities/中的诊断工具
3. **查看文档** - 到docs/目录查找相关说明
4. **自定义样式** - 在css/目录中修改或添加

## 🚀 快速开始

1. **配置API** → 打开 `API池管理工具.html`
2. **挖掘爆款** → 打开 `01 低粉爆款挖掘 .html`
3. **管理博主** → 打开 `02 YouTube博主管理工具.html`
4. **监控视频** → 打开 `对标监控/chuhai.html`
