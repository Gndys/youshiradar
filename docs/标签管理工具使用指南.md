# 标签管理工具使用指南

## 📋 功能概述

标签管理工具是 PHANTOM RADAR 工具集的核心组件之一，提供了统一、高效的标签管理解决方案。通过这个工具，你可以：

- 📊 **查看所有标签** - 一目了然地查看所有标签及其使用情况
- ✏️ **重命名标签** - 统一修改标签名称，自动更新所有关联博主
- 🔗 **合并标签** - 将多个相似标签合并为一个，优化标签体系
- 🗑️ **删除标签** - 批量删除不需要的标签
- 👀 **查看关联** - 快速查看使用某个标签的所有博主
- 📈 **统计分析** - 实时统计标签使用情况

## 🚀 快速开始

### 1. 访问标签管理工具

从主页点击 **🏷️ 管理标签** 按钮，或在导航栏选择 **标签管理**。

### 2. 同步标签数据

首次使用时，点击 **🔄 同步博主标签** 按钮，从博主管理工具中同步所有标签。

## 📖 核心功能详解

### 查看标签列表

进入页面后，你会看到：
- **标签卡片** - 每个标签显示为一个卡片，包含标签名称和使用次数
- **统计面板** - 顶部显示总标签数、总使用次数、平均使用等统计信息
- **排序选项** - 支持按名称或使用次数排序

### 搜索和筛选

1. **搜索标签**
   - 在搜索框输入关键词，实时筛选匹配的标签
   - 支持模糊搜索

2. **排序标签**
   - 名称 A-Z / Z-A
   - 使用次数从高到低 / 从低到高

### 创建新标签

1. 在顶部输入框输入标签名称
2. 点击 **➕ 创建标签** 按钮
3. 新标签将出现在列表中（初始使用次数为0）

### 重命名标签

1. 在标签卡片上点击 **✏️ 重命名** 按钮
2. 在弹出窗口输入新的标签名称
3. 点击 **确认**
4. 系统将自动更新所有使用该标签的博主

**注意事项：**
- 新名称不能与现有标签重复
- 重命名会立即同步到所有相关博主
- 操作不可撤销，请谨慎操作

### 查看使用该标签的博主

1. 点击标签卡片上的 **👀 查看博主** 按钮
2. 展开后显示所有使用该标签的博主列表
3. 包含博主头像和名称
4. 再次点击可收起列表

### 删除单个标签

1. 点击标签卡片上的 **🗑️ 删除** 按钮
2. 确认删除操作
3. 标签将从所有博主中移除

### 批量操作

#### 批量选择
1. 勾选标签卡片左上角的复选框
2. 可点击 **全选** 按钮选择所有标签
3. 已选择的标签会高亮显示
4. 底部会弹出批量操作栏

#### 批量删除
1. 选择要删除的多个标签
2. 点击底部操作栏的 **🗑️ 删除标签** 按钮
3. 确认删除
4. 所选标签将从所有博主中移除

#### 批量合并
1. 选择至少2个要合并的标签
2. 点击底部操作栏的 **🔗 合并标签** 按钮
3. 在弹出窗口中：
   - **选择已有标签** - 从下拉列表选择目标标签
   - **或输入新标签名** - 创建新标签作为合并目标
4. 点击 **确认合并**
5. 所有使用被合并标签的博主将自动更新为目标标签

**合并示例：**
- 原标签：`科技`、`tech`、`技术`
- 合并为：`科技` 
- 结果：所有使用 `tech` 和 `技术` 的博主改为使用 `科技` 标签

### 导出标签数据

1. 点击 **📤 导出标签** 按钮
2. 系统生成 CSV 文件，包含：
   - 标签名称
   - 使用次数
   - 关联的博主列表
3. 文件自动下载到本地

## 💡 使用技巧

### 1. 定期整理标签

建议每月整理一次标签：
- 合并相似标签（如 `tech` 和 `科技`）
- 删除不再使用的标签
- 统一命名规范

### 2. 建立标签体系

建议的标签分类：
- **内容类型**：教程、评测、开箱、vlog 等
- **领域分类**：科技、美食、旅游、游戏等
- **风格定位**：搞笑、专业、情感、知识等
- **受众群体**：学生、职场、家长等
- **自定义标签**：根据你的需求创建

### 3. 标签命名规范

推荐的命名规范：
- 使用简洁、明确的名称
- 避免特殊字符
- 中英文统一（选择一种语言）
- 使用小写或首字母大写保持一致

### 4. 批量合并技巧

合并标签时的最佳实践：
1. 先查看每个标签的使用情况
2. 选择使用次数最多的作为目标标签
3. 或创建一个更规范的新标签名

### 5. 与博主管理配合使用

标签管理工具与博主管理工具紧密集成：
- 在博主管理中添加标签
- 在标签管理中统一整理
- 修改会实时同步到博主数据

## ⚠️ 注意事项

1. **数据同步**
   - 标签数据从博主管理工具同步
   - 所有修改会实时保存到 IndexedDB
   - 修改标签会自动更新相关博主

2. **操作不可撤销**
   - 删除和合并操作无法撤销
   - 建议操作前先导出数据备份
   - 重要操作会有二次确认

3. **标签冲突**
   - 重命名时不能使用已存在的标签名
   - 合并时目标标签可以是新名称

4. **性能优化**
   - 标签数量建议控制在 100 个以内
   - 过多标签会影响筛选效率
   - 定期整理可提升使用体验

## 🔧 常见问题

### Q: 标签列表为空？
**A:** 点击 **🔄 同步博主标签** 按钮，从博主管理工具同步数据。

### Q: 修改标签后博主数据没更新？
**A:** 修改会自动保存，刷新博主管理页面即可看到更新。

### Q: 可以批量导入标签吗？
**A:** 目前通过博主管理工具的标签功能添加，然后在标签管理中整理。

### Q: 删除标签会影响博主数据吗？
**A:** 只会移除标签，不会删除博主本身，博主的其他数据不受影响。

### Q: 合并标签时如何选择目标？
**A:** 建议选择使用最广泛或最规范的标签名作为目标。

### Q: 可以恢复已删除的标签吗？
**A:** 无法直接恢复，建议操作前先导出数据备份。

## 📊 最佳实践案例

### 场景1：统一多语言标签

**问题：** 同一概念有多个不同的标签
- `tech`, `科技`, `Technology`, `技术`

**解决方案：**
1. 选择这4个标签
2. 点击批量合并
3. 合并为统一的 `科技` 标签
4. 所有相关博主自动更新

### 场景2：按领域分类整理

**目标：** 建立清晰的领域标签体系

**步骤：**
1. 导出当前所有标签，分析现状
2. 制定标签分类标准（如：科技、美食、旅游等）
3. 创建标准标签名称
4. 批量合并相似标签到标准名称
5. 删除不再需要的标签

### 场景3：定期维护

**建议流程：**

**每周：**
- 在博主管理中为新博主添加标签

**每月：**
1. 打开标签管理工具
2. 检查使用次数为0的标签，考虑删除
3. 查找相似标签，进行合并
4. 导出标签数据备份

**每季度：**
- 重新审视标签体系，优化分类
- 与团队讨论标签命名规范
- 更新标签使用指南
