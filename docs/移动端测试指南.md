# 移动端优化测试指南

## 快速测试清单

### 1. 首页导航测试（index.html）

**测试步骤：**
1. 在移动设备打开 `index.html`
2. 点击右上角的汉堡菜单图标（☰）
3. 验证菜单是否展开
4. 点击任一菜单项
5. 验证菜单是否自动关闭

**预期结果：**
- ✅ 菜单平滑展开，带有动画效果
- ✅ 点击图标变为关闭符号（✕）
- ✅ 菜单项清晰可见，易于点击
- ✅ 点击菜单项或外部区域后菜单自动关闭

### 2. 按钮布局测试（工具页面）

**测试步骤：**
1. 打开 `01 低粉爆款挖掘.html`
2. 向下滚动到按钮组区域
3. 观察按钮的排列方式

**预期结果：**
- ✅ "开始分析"按钮占据整行（全宽）
- ✅ 其他按钮以两列形式排列
- ✅ 按钮高度至少44px，易于点击
- ✅ 按钮之间有适当间距，不会误触

### 3. 复制功能测试

**测试步骤：**
1. 在 `01 低粉爆款挖掘.html` 中搜索视频
2. 点击"获取博主链接"
3. 点击"📋 复制全部"按钮
4. 查看是否显示成功提示

**预期结果：**
- ✅ 显示"✅ 已复制 X 个链接"的成功提示
- ✅ 如果失败，显示"❌ 复制失败，请手动复制"
- ✅ iOS Safari 可以正常复制
- ✅ Android Chrome 可以正常复制

**测试 API 池工具：**
1. 打开 `API池管理工具.html`
2. 点击任意 API Key 的"📋 复制完整Key"
3. 验证复制功能

### 4. API 初始化测试

**测试步骤：**
1. 清除浏览器缓存和数据
2. 打开开发者工具控制台（如果可能）

**预期结果：**
- ✅ 控制台显示初始化日志：
  ```
  🚀 开始初始化应用...

  📦 API池加载完成，当前数量: X
  ```

### 5. 输入框缩放测试（iOS）

**测试步骤：**

2. 点击任意输入框（如搜索关键词）

- ✅ 输入框获得焦点时，页面不会自动缩放

## 常见问题排查

### 问题1：导航菜单不展开

**排查步骤：**
1. 检查浏览器控制台是否有JavaScript错误
2. 确认使用的是最新版本的文件
3. 尝试硬刷新（Ctrl+Shift+R 或 Cmd+Shift+R）

### 问题2：复制功能失败

**可能原因：**
- 浏览器不支持 Clipboard API
- 页面非 HTTPS 环境（某些浏览器限制）
- 浏览器安全设置过严

**解决方案：**
- 使用最新版浏览器
- 如果是本地文件，尝试使用本地服务器
- 检查浏览器权限设置

### 问题3：API初始化失败

**排查步骤：**
1. 查看控制台错误信息
2. 检查 IndexedDB 是否被禁用
3. 清除浏览器缓存后重试
4. 检查网络连接

**解决方法：**
- 启用浏览器的 IndexedDB 功能
- 退出隐私/无痕模式
- 手动添加 API Key

## 性能测试

### 加载速度
- **首次加载：** 应在 3 秒内完成初始化
- **后续加载：** 应在 1 秒内恢复状态

### 响应速度
- **按钮点击：** 应即时响应（< 100ms）
- **菜单展开：** 应流畅展开（300ms 动画）
- **数据搜索：** 取决于 API 响应时间
