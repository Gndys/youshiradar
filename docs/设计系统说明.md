# YouTube工具设计系统 v4.0

## 🎨 设计哲学

这不是一次简单的UI美化，而是**完整的视觉系统重构**。

### 核心原则

1. **克制的配色** - 摒弃刺眼的渐变，采用精心调校的色彩系统
2. **精确的间距** - 基于8px网格，确保视觉呼吸感
3. **微妙的交互** - 提升体验而不喧宾夺主
4. **清晰的层级** - 让用户专注于内容，而非装饰

---

## 🏗️ 系统架构

### 色彩系统

#### 主色调 - 克制的紫色
```css
--primary-500: #8b5cf6  /* 主要操作 */
--primary-600: #7c3aed  /* 悬停状态 */
--primary-700: #6d28d9  /* 按下状态 */
```

**为什么是紫色？**
- 比蓝色更有个性，比红色更克制
- 象征创造力和专业性
- 在深色和浅色背景下都有良好表现

#### 灰度系统 - 50种灰
```css
--gray-50: #fafafa   /* 背景 */
--gray-900: #171717  /* 文本 */
```

**每个灰度都经过精心调校**
- 确保足够的对比度（符合WCAG AA标准）
- 在不同显示器上一致的表现
- 支持深色模式切换

#### 语义色彩
```css
--success-500: #10b981  /* 成功 - 温和的绿色 */
--error-500: #ef4444    /* 错误 - 清晰但不刺眼的红色 */
--warning-500: #f59e0b  /* 警告 - 琥珀色 */
```

### 间距系统 - 8px网格

```css
--space-2: 0.5rem   /* 8px  - 最小间距 */
--space-4: 1rem     /* 16px - 标准间距 */
--space-6: 1.5rem   /* 24px - 组件间距 */
--space-8: 2rem     /* 32px - 区块间距 */
```

**为什么是8px？**
- 符合人眼视觉规律
- 易于缩放和计算
- 适配主流屏幕分辨率

### 圆角系统

```css
--radius-sm: 6px   /* 小元素：标签、徽章 */
--radius-md: 8px   /* 按钮、输入框 */
--radius-lg: 12px  /* 卡片 */
--radius-xl: 16px  /* 大型容器 */
```

**圆角的心理学**
- 小圆角（6-8px）：现代、专业
- 大圆角（16-24px）：友好、轻松
- 我们选择中等圆角，兼顾专业与易用

### 阴影系统

```css
--shadow-sm: 0 1px 3px 0 rgb(0 0 0 / 0.1)      /* 悬浮感 */
--shadow-md: 0 4px 6px -1px rgb(0 0 0 / 0.1)   /* 轻微层次 */
--shadow-xl: 0 20px 25px -5px rgb(0 0 0 / 0.1) /* 强烈层次 */
```

**阴影的物理学**
- 模拟真实光源
- Y轴偏移 > X轴偏移（光从上方照射）
- 模糊半径适中（过度模糊显得廉价）

### 字体系统

#### 字体栈
```css
--font-sans: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Noto Sans', Helvetica, Arial;
```

**为什么这样排序？**
1. 系统字体优先（性能最佳）
2. 跨平台一致性
3. 优雅降级

#### 字重层级
```css
--font-normal: 400    /* 正文 */
--font-medium: 500    /* 次要标题 */
--font-semibold: 600  /* 主要标题 */
--font-bold: 700      /* 强调 */
```

**字重的语义**
- 避免使用极端字重（100, 900）
- 确保至少3级层级
- 在小尺寸下，字重差异更明显

### 动画系统

```css
--transition-fast: 150ms    /* 即时反馈 */
--transition-base: 200ms    /* 标准交互 */
--transition-slow: 300ms    /* 重要变化 */
```

**动画的节奏**
- 150ms：几乎察觉不到，但提供反馈
- 200ms：标准交互，不快不慢
- 300ms：吸引注意力

#### 缓动函数
```css
--ease-out: cubic-bezier(0, 0, 0.2, 1)  /* 开始快，结束慢 */
```

**为什么用ease-out？**
- 符合物理直觉
- 让交互感觉更有重量感
- iOS和Material Design的默认选择

## 🧩 组件设计决策

### 按钮

#### 主按钮（Primary）
```css
background: var(--primary-600);
border: 1px solid var(--primary-600);
```

**设计决策：**
- 使用600而非500：在白色背景上对比度更好
- 1px边框：定义边界，避免颜色溢出
- 悬停时Y轴上移1px + 阴影：提供物理反馈

#### 次级按钮（Secondary）
```css
background: var(--bg-primary);
border: 1px solid var(--border-primary);
```

**设计决策：**
- 透明背景 + 边框：让主按钮更突出
- 悬停时改变背景色而非边框色：更微妙的反馈

### 卡片

```css
.video-card {
  border-radius: var(--radius-xl);  /* 16px */
  border: 1px solid var(--border-primary);
  box-shadow: var(--shadow-sm);
}

.video-card:hover {
  transform: translateY(-4px);
  box-shadow: var(--shadow-2xl);
}
```

**设计决策：**
- 默认状态：微妙的阴影，建立层次
- 悬停状态：上移4px + 大阴影，强烈的交互反馈
- 4px而非2px：在视频卡片这种大元素上，需要明显的位移

### 输入框

```css
input:focus {
  border-color: var(--primary-500);
  box-shadow: 0 0 0 3px rgba(139, 92, 246, 0.1);
}
```

**设计决策：**
- 3px光环：比2px更明显，比4px不夸张
- 10%透明度：提供焦点指示但不喧宾夺主
- 无过渡动画：焦点应该是即时的

## 🌓 深色模式

### 色彩调整策略

#### 背景
```css
/* 浅色 */
--bg-primary: #ffffff
--bg-secondary: #fafafa

/* 深色 */
--bg-primary: #0a0a0a
--bg-secondary: #171717
```

**为什么不是纯黑？**
- 纯黑在OLED屏上过于刺眼
- 深灰提供更好的层次感
- 减少眼疲劳

#### 阴影增强
```css
/* 深色模式下阴影透明度增加 */
--shadow-sm: 0 1px 3px 0 rgb(0 0 0 / 0.4)  /* 0.1 → 0.4 */
```

**为什么加深阴影？**
- 深色背景上，浅色阴影不明显
- 需要更强对比度来建立层次

## 📱 响应式设计

### 断点策略

```css
@media (max-width: 768px) {
  /* 移动端优化 */
}
```

**为什么只有一个断点？**
- 遵循"移动优先"原则
- 768px是平板和手机的分水岭
- 简化维护成本

### 移动端适配

```css
@media (max-width: 768px) {
  .btn {
    width: 100%;  /* 按钮全宽 */
  }
  
  .button-group {
    flex-direction: column;  /* 垂直堆叠 */
  }
}
```

**移动端哲学：**
- 按钮要够大（最小44×44px）
- 减少横向滚动
- 优先垂直布局

## ♿ 可访问性

### 对比度

所有文本都符合WCAG AA标准：
- 正文：4.5:1
- 大文本：3:1

### 焦点可见性

```css
*:focus-visible {
  outline: 2px solid var(--primary-500);
  outline-offset: 2px;
}
```

**设计决策：**
- 2px外轮廓：清晰可见
- 2px偏移：与元素保持间距
- 不使用:focus：避免鼠标点击时的轮廓

### 减少动画

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

**为什么不是0？**
- 0可能导致某些JS动画库出错
- 0.01ms在人眼看来等同于无动画

## 🔧 使用指南

### 快速开始

1. **引入设计系统**
```html
<link rel="stylesheet" href="design-system.css">
```

2. **移除旧样式**
删除所有内联的`<style>`标签中的基础样式

3. **使用变量**
```css
/* ❌ 不要硬编码 */
color: #8b5cf6;

/* ✅ 使用变量 */
color: var(--primary-500);
```

### 命名规范

#### BEM方法论
```html
<!-- Block -->
<div class="video-card">
  <!-- Element -->
  <div class="video-card__thumbnail">
    <!-- Modifier -->
    <span class="video-card__badge video-card__badge--hot">🔥</span>
  </div>
</div>
```

**为什么用BEM？**
- 清晰的命名层级
- 避免样式冲突
- 易于维护

#### 语义化类名
```html
<!-- ❌ 不好 -->
<button class="red-button">删除</button>

<!-- ✅ 好 -->
<button class="btn btn-danger">删除</button>
```

## ❓ 常见问题

### Q: 为什么不用CSS-in-JS？
A: 
- 性能开销
- 构建复杂度
- 纯CSS更易维护

### Q: 能否自定义主色？
A: 
可以，修改`--primary-*`变量即可：
```css
:root {
  --primary-500: #your-color;
  --primary-600: #your-darker-color;
}
```

### Q: 如何禁用深色模式？
A: 
删除`[data-theme="dark"]`相关样式即可

### Q: 变量在IE11中不工作？
A: 
IE11不支持CSS变量，建议使用PostCSS降级：
```js
postcss([
  require('postcss-css-variables')()
])
```
